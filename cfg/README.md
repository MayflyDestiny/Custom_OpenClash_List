# Config Templates (配置模板)

本目录提供基于 OpenClash 的完整配置模板，预设了科学的分流策略和策略组体系。

## 📄 模板介绍

### 1. `Custom_Clash_Smart_Emoji.yaml`
**Emoji 风格配置**
- **特点**：策略组名称使用 Emoji 图标（如 `🚀 节点选择`, `📺 流媒体`）。
- **优点**：通用性强，不依赖外部图片资源，加载速度快，在任何客户端都能正常显示。
- **适用**：喜欢简洁风格或网络环境加载图片较慢的用户。

### 2. `Custom_Clash_Smart_Icon.yaml`
**图标风格配置**
- **特点**：策略组引用了 `IconSet` 目录下的精美图片图标。
- **优点**：视觉效果极佳，界面美观度高。
- **依赖**：需要客户端支持 `icon` 字段，且需要网络加载图标（建议配置 CDN）。
- **适用**：追求美观和个性化的用户。

## 🛠️ 核心功能

两款模板均包含以下高级特性：

### 🔍 智能锚点 (Anchors)
- **BaseFB (Fallback)**: 自动回退，选择可用的节点。
- **BaseSL (Select)**: 手动选择。
- **BaseSmart**: 智能负载均衡（部分支持）。

### 🌍 自动地区筛选 (Regex Filters)
预置了强大的正则表达式，自动将订阅中的节点按地区分组：
- **FilterHK**: 香港节点
- **FilterJP**: 日本节点
- **FilterUS**: 美国节点
- **FilterSG**: 新加坡节点
- **FilterTW**: 台湾节点
- ...以及欧盟、英国等。

### 🚦 预设策略组
- **ChatAI**: 针对 ChatGPT, Claude 等 AI 服务的策略优化（优先美国节点）。
- **Stream**: 流媒体策略组。
- **Games**: 游戏策略组（默认直连或低延迟节点）。
- **GitHub / Social**: 针对特定服务的策略组。

## 📝 使用指南

1. **选择模板**：根据喜好选择 Emoji 版或 Icon 版。
2. **修改订阅**：
   打开文件，找到 `proxy-providers` 部分，将 `url` 替换为您的机场订阅链接。
   ```yaml
   proxy-providers:
     机场名称:
       ...
       url: '您的订阅链接'
   ```
3. **上传配置**：将修改后的 `.yaml` 文件上传至 OpenClash 的配置管理中并启动。
